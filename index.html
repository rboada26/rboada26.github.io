Featured Projects

L1es

L1ES is a comprehensive cache simulation and security analysis tool I developed in Rust to explore how cache design decisions impact vulnerability to timing-based side-channel attacks. The simulator implements multiple cache architectures (direct-mapped, set-associative, fully associative) with configurable parameters and demonstrates four major attack classes: Prime+Probe cache set monitoring, Flush+Reload shared memory exploitation, Spectre speculative execution vulnerabilities, and Meltdown out-of-order execution privilege bypass. Built as a production-quality command-line tool with comprehensive CLI interface, the application generates quantitative research data through CSV/JSON export, enabling systematic comparison of attack effectiveness across different cache configurations and providing educational value for understanding the security implications of microarchitectural design trade-offs.

Hardware Sice Channel Attacks (In progress)

I will build a side-channel power-analysis lab that instruments an FPGA running a lightweight AES-128 core, captures thousands of aligned power traces using a shunt resistor and a USB oscilloscope (or ADC), and implements a Correlation Power Analysis (CPA) pipeline in Python to recover AES key bytes. Through this project I will develop practical skills with RTL/FPGA toolchains (Verilog + yosys/nextpnr or Vivado), measurement and data-acquisition, trace preprocessing, and statistical attack techniques, and I will publish reproducible artifacts—trace datasets, collection and analysis scripts, and detailed documentation—for verification. My goal is to quantify real-world leakage and attack success under realistic noise conditions, evaluate simple countermeasures, and demonstrate my preparation and fit for graduate research in hardware security.
